\ssc{1-1-1 專題整體介紹與說明⽂案與 1-1-2 功能設計}
《Beat Runner》是一款結合下落式音樂遊戲與滑鼠角色控制的創新遊戲。開始介面的部分有三個按鍵 — Settings 按鍵可以調整遊戲的音量大小、Songs 按鍵可以選擇遊玩的歌曲，其中包括適合新手的 Thefatrat 的 Unity，在遊戲中顯示為《Tmp》，以及難度較高的 Sakuzyo 的《Destr0yer》，而按下 Start 則會開始遊戲。

開始遊戲之後，首先會進入遊戲難度的調整，包含 Lanes 軌道數的調整、Fragments 可見列數的調整、以及 Mods 遊戲模組的調整。Lanes 可以調整遊戲的軌道數，但由於我們的譜面並非隨機生成而是連結外部、我們自己寫的 .txt 檔案，時間關係並沒有寫 Lanes = 4 以外的譜面。Fragments 部分則可以調整可見列數，數值越大，可以越早看見 Notes 的出現，但音符更窄故可能較難判斷與判定線的距離。Mods 可以選擇不同模式的康威生命遊戲（Conway's Game of Life）模組，遊戲中將會在 TapNote 上執行相應的康威生命遊戲，但目前結算時的準確度不會根據之調整分母。選擇完畢後會倒數 3 秒，以供玩家準備。

遊戲中的下落式的音符可分為兩種，第一種是由鍵盤操作的 Key note，又細分為紅色的 Tap note 與綠色的 Hold note。Tap note 玩家必須在正確的節奏點按下對應的按鍵，Hold note 玩家則須在期間內持續按住對應的按鍵以持續得分。以軌道數 Lanes = 4 為例，對應的按鍵即為 A、S、D、F ，若軌道數目更多則依序對應 G、H 等右方的鍵盤按鍵。Tap note 根據玩家按下對應按鍵的時機，評分判定分為五個等級—— Perfect、Great、Good、Bad、Miss，分別能獲得 100\%、75\%、50\%、25\%、0\% 的分數，滿分為 1000 分。持續避免 Bad 或 Miss 可以累積 Combo 數。

第二種是是由滑鼠操縱角色碰觸的 Mouse note，分為紅色與綠色兩種，碰觸綠色音符可以持續獲得分數，最多 400 分。相反地，紅色的音符則會造成等量的扣分。滑鼠左右操作可以使角色在白色線上移動，讓玩家得以控制角色在觸碰綠色音符的同時避開紅色音符。遊戲過程中，鍵盤的操作與滑鼠的移動必須同時進行，考驗玩家的左右手協調與反應力。

在遊戲中，你可以按下 ESC 進入暫停畫面，在這裡你可以回到遊戲、開啟新遊戲與離開遊戲。遊戲結束後玩家可以得知得分、最高連擊數、準確度，以及各個判定標準的數量，玩家可以選擇重試或回到主選單。在遊戲的任何時候，都可以用 Ctrl + W 退出遊戲。

void gameOfLifeSettings(SDL_Renderer *renderer, TTF_Font *font, int screenWidth,
                        int screenHeight) {
  mystd::array<uint16_t, 2> tempRules = currentRules;

  const int buttonSize = 40;
  const int buttonSpacing = 5;
  const int rowSpacing = 60;

  int totalWidth = 9 * buttonSize + 8 * buttonSpacing;
  int startX = (screenWidth - totalWidth) / 2;
  int row1Y = screenHeight / 2 - rowSpacing;
  int row2Y = screenHeight / 2 + rowSpacing;

  const int okButtonWidth = 100;
  const int okButtonHeight = 50;
  int okButtonX = (screenWidth - okButtonWidth) / 2;
  int okButtonY = row2Y + rowSpacing * 2;

  bool settingsRunning = true;
  bool redrawNeeded = true;

  while (settingsRunning) {
    SDL_Event event;
    while (SDL_PollEvent(&event)) {
      if (event.type == SDL_QUIT) {
        settingsRunning = false;
        break;
      } else if (event.type == SDL_KEYDOWN) {
        if (event.key.keysym.sym == SDLK_RETURN ||
            event.key.keysym.sym == SDLK_ESCAPE) {
          currentRules = tempRules;
          settingsRunning = false;
        }
      } else if (event.type == SDL_MOUSEBUTTONDOWN) {
        if (event.button.button == SDL_BUTTON_LEFT) {
          int mouseX = event.button.x;
          int mouseY = event.button.y;

          for (int i = 0; i < 9; ++i) {
            int buttonX = startX + i * (buttonSize + buttonSpacing);
            SDL_Rect buttonRect = {buttonX, row1Y, buttonSize, buttonSize};

            if (mouseX >= buttonRect.x &&
                mouseX <= buttonRect.x + buttonRect.w &&
                mouseY >= buttonRect.y &&
                mouseY <= buttonRect.y + buttonRect.h) {
              tempRules[0] ^= (1 << i);
              redrawNeeded = true;
            }
          }

          for (int i = 0; i < 9; ++i) {
            int buttonX = startX + i * (buttonSize + buttonSpacing);
            SDL_Rect buttonRect = {buttonX, row2Y, buttonSize, buttonSize};

            if (mouseX >= buttonRect.x &&
                mouseX <= buttonRect.x + buttonRect.w &&
                mouseY >= buttonRect.y &&
                mouseY <= buttonRect.y + buttonRect.h) {
              tempRules[1] ^= (1 << i);
              redrawNeeded = true;
            }
          }

          SDL_Rect okRect = {okButtonX, okButtonY, okButtonWidth,
                             okButtonHeight};
          if (mouseX >= okRect.x && mouseX <= okRect.x + okRect.w &&
              mouseY >= okRect.y && mouseY <= okRect.y + okRect.h) {
            currentRules = tempRules;
            settingsRunning = false;
          }
        }
      }
    }

    if (redrawNeeded) {
      SDL_SetRenderDrawColor(renderer, 30, 30, 40, 255);
      SDL_RenderClear(renderer);

      for (int i = 0; i < 9; ++i) {
        int buttonX = startX + i * (buttonSize + buttonSpacing);
        SDL_Rect buttonRect = {buttonX, row1Y, buttonSize, buttonSize};

        bool enabled = (tempRules[0] >> i) & 0b1;

        if (enabled) {
          SDL_SetRenderDrawColor(renderer, 100, 200, 100, 255);
        } else {
          SDL_SetRenderDrawColor(renderer, 200, 100, 100, 255);
        }
        SDL_RenderFillRect(renderer, &buttonRect);

        SDL_SetRenderDrawColor(renderer, 255, 255, 255, 255);
        SDL_RenderDrawRect(renderer, &buttonRect);

        SDL_Color numColor = {255, 255, 255, 255};
        renderText(renderer, std::to_string(i), buttonX + buttonSize / 2,
                   row1Y + buttonSize / 2, font, numColor, ALIGN_CENTER);
      }

      for (int i = 0; i < 9; ++i) {
        int buttonX = startX + i * (buttonSize + buttonSpacing);
        SDL_Rect buttonRect = {buttonX, row2Y, buttonSize, buttonSize};

        bool enabled = (tempRules[1] >> i) & 0b1;

        if (enabled) {
          SDL_SetRenderDrawColor(renderer, 100, 200, 100, 255);
        } else {
          SDL_SetRenderDrawColor(renderer, 200, 100, 100, 255);
        }
        SDL_RenderFillRect(renderer, &buttonRect);

        SDL_SetRenderDrawColor(renderer, 255, 255, 255, 255);
        SDL_RenderDrawRect(renderer, &buttonRect);
        SDL_Color numColor = {255, 255, 255, 255};
        renderText(renderer, std::to_string(i), buttonX + buttonSize / 2,
                   row2Y + buttonSize / 2, font, numColor, ALIGN_CENTER);
      }

      SDL_Rect okRect = {okButtonX, okButtonY, okButtonWidth, okButtonHeight};
      SDL_SetRenderDrawColor(renderer, 100, 150, 255, 255);
      SDL_RenderFillRect(renderer, &okRect);
      SDL_SetRenderDrawColor(renderer, 255, 255, 255, 255);
      SDL_RenderDrawRect(renderer, &okRect);

      SDL_Color textColor = {255, 255, 255, 255};
      std::string surviveText = "Survive with neighbors:";
      std::string reviveText = "Revive with neighbors:";

      int surviveTextX = screenWidth / 2;
      int surviveTextY = row1Y - 40;
      renderText(renderer, surviveText, surviveTextX, surviveTextY, font,

      int reviveTextX = screenWidth / 2;
      int reviveTextY = row2Y - 40;

      renderText(renderer, okText, okTextX, okTextY, font, textColor,
                 ALIGN_CENTER);

      SDL_RenderPresent(renderer);
      redrawNeeded = false;
    }
  }
}

\item iterator proxy 類別間的互通。
\item 避免破壞 class 封裝性或擴大 public 介面。

\item \verb|circulate| 容器本身宣告 \verb|friend bool operator==(const circulate \&lhs, const circulate \&rhs)| 以允許直接比較兩個 \verb|circulate| 物件之內部狀態。  由於此等比較運算屬於容器語意的一部分，將其實作為非成員 friend 函式可維持與標準容器一致的使用方式。（line~333）
\item \verb|include/tuple.hpp| 中，
\verb|tuple| 類別使用 \verb|friend| 宣告實作

\end{itemize}
\ssc{2-A-3-5 class composition}
\sssc{Resource Acquisition Is Initialization (RAII)}
\label{RAII}
RAII is a programming idiom in C++ used to manage resource lifetimes automatically. The key idea is:
\begin{itemize}
    \item Resources (e.g., memory, file handles, sockets) are acquired in the constructor of a class.
    \item Resources are released in the destructor, ensuring deterministic cleanup and exception safety.
\end{itemize}
\sssc{專題使用}
本專題中，多數 class 採用 \textbf{composition（組合）} 方式建立內部關係，其核心目標包括：
\begin{itemize}
\item 將資料與功能封裝於物件內部，避免 global state 干擾。
\item 透過明確的成員關係管理所有權與生命週期，達成 \hyperref[raii]{\textbf{Resource Acquisition Is Initialization (RAII)}}。
\item 與 C++20 標準及 STL 慣例保持一致。
\end{itemize}
具體實例請參見 \hyperref[2-A-3-1]{\textbf{2-A-3-1 public 變數使用與原因}}、\hyperref[2-A-3-2]{\textbf{2-A-3-2 private 變數使用與原因}}、\hyperref[2-A-3-3]{\textbf{2-A-3-3 protected 變數使用與原因}} 與 \hyperref[2-B-3-1-1]{\textbf{2-B-3-1-1 指出組裝關係}}。
\ssc{2-B-1-1 建構子、複製建構子、解構子}
\sssc{include/vector.hpp}
\begin{itemize}
\item \verb|constexpr vector() noexcept(noexcept(Allocator()))| (line~95)：預設建構子，建立空 vector。

\ssc{2-B-1-2 指標成員處理}
\label{2-B-1-2}
\sssc{iterator 成員處理一併說明}
本報告於「指標成員處理」章節中一併說明「iterator 成員處理」，理由如下：
\begin{itemize}
\item 在實作層面上，iterator 本質上即為一種受控的指標抽象（pointer abstraction），其內部通常直接持有原始指標，或持有可間接定位至元素之指標型成員（如 \verb|T*|、container reference 加 index 等）。iterator 的比較運算、位移運算與解參考操作（\verb|operator*|、\verb|operator->|、\verb|operator+| 等），均直接依賴內部指標或等價結構，其正確實作本質上即為指標操作的安全封裝。
\item iterator 的正確性與安全性，直接取決於其所持有之指標成員的生命週期、有效性與邊界管理方式，並承擔以下責任：
  \begin{itemize}
  \item 描述元素存取位置（position semantics）。
  \item 維持與底層容器狀態的一致性。
  \item 在不暴露底層指標的前提下，提供 STL 風格的介面。
  \end{itemize}
\end{itemize}
\sssc{Exception Safety Guarantees}
\label{exception}
STL containers follow strict \textbf{exception safety guarantees} during operations such as insertion, deletion, or reallocation:
\begin{itemize}
    \item \textbf{No-throw guarantee:} The operation never throws an exception.
    \item \textbf{Strong guarantee:} If an exception occurs, the container's state remains unchanged.
    \item \textbf{Basic guarantee:} Some changes may occur, but the container remains in a valid state.
\end{itemize}
In my implementation, I use \texttt{try-catch} blocks and proper deallocation to handle exceptions, prefer no-throwing branches when possible, and provide explicit \texttt{noexcept} specifications.

        elems = mystd::allocator_traits<Allocator>::allocate(alloc, count);
        sz = cap = count;
        if constexpr (std::is_trivially_copyable_v<T>) {
          if (std::contiguous_iterator<InputIt>)
            std::memmove(elems, &*first, count * sizeof(T));
          else
            mystd::copy(first, last, elems);
        } else {
          std::size_t i = 0;
          try {
            for (; first != last; ++first, ++i)
              mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                            *first);
          } catch (...) {
            for (std::size_t j = 0; j < i; ++j)
              mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
            deallocate();
            throw;
          }
        }
      }
    } else
      for (; first != last; ++first)
        push_back(*first);
  }

  constexpr vector(const vector &other)
      : alloc(mystd::allocator_traits<

      elems = mystd::allocator_traits<Allocator>::allocate(alloc, cap);
      if constexpr (std::is_trivially_copyable_v<T>)

                                                          other.elems[i]);
        } catch (...) {
          for (std::size_t j = 0; j < i; ++j)
            mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
          mystd::allocator_traits<Allocator>::deallocate(alloc, elems, cap);

        }
      }
    }
  }

  constexpr vector(vector &&other) noexcept(
      mystd::allocator_traits<
          Allocator>::propagate_on_container_move_assignment::value ||
      mystd::allocator_traits<Allocator>::is_always_equal::value)
      : alloc(std::move(other.alloc)), elems(nullptr), sz(0), cap(0) {
    if constexpr (mystd::allocator_traits<Allocator>::
                      propagate_on_container_move_assignment::value ||
                  mystd::allocator_traits<Allocator>::is_always_equal::value ||
                  (requires {
                    { alloc == other.alloc } -> std::convertible_to<bool>;
                  } && (alloc == other.alloc))) {
      elems = std::exchange(other.elems, nullptr);
      sz = std::exchange(other.sz, 0);
      cap = std::exchange(other.cap, 0);
    } else {
      if (other.sz > 0) {
        elems = mystd::allocator_traits<Allocator>::allocate(alloc, other.sz);
        sz = cap = other.sz;
        if constexpr (std::is_trivially_move_constructible_v<T>)
          std::memmove(elems, other.elems, sz * sizeof(T));
        else if (std::is_nothrow_move_constructible_v<T> ||
                 !std::is_copy_constructible_v<T>) {
          std::size_t i = 0;
          try {
            for (; i < sz; ++i)
              mystd::allocator_traits<Allocator>::construct(
                  alloc, elems + i, std::move(other.elems[i]));
          } catch (...) {
            for (std::size_t j = 0; j < i; ++j)
              mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
            mystd::allocator_traits<Allocator>::deallocate(alloc, elems, cap);
            throw;
          }
        } else {
          std::size_t i = 0;
          try {
            for (; i < sz; ++i)
              mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                            other.elems[i]);
          } catch (...) {
            for (std::size_t j = 0; j < i; ++j)
              mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
            mystd::allocator_traits<Allocator>::deallocate(alloc, elems, cap);
            throw;
          }
        }
        if constexpr (!std::is_trivially_destructible_v<T> &&
                      !std::is_trivially_move_constructible_v<T>) {
          if (std::is_nothrow_move_constructible_v<T> ||
              !std::is_copy_constructible_v<T>) {
            for (std::size_t j = 0; j < sz; ++j)
              mystd::allocator_traits<Allocator>::destroy(alloc,
                                                          other.elems + j);
          }
        }
        other.elems = nullptr;
        other.sz = other.cap = 0;
      }
    }
  }

  constexpr ~vector() {
    if (elems) {
      if constexpr (!std::is_trivially_destructible_v<T>)
        for (std::size_t i = 0; i < sz; ++i)
          mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
      mystd::allocator_traits<Allocator>::deallocate(alloc, elems, cap);
    }
  }

  constexpr vector &operator=(const vector &other) {
    if (this != &other) {
      if constexpr (mystd::allocator_traits<Allocator>::
                        propagate_on_container_copy_assignment::value &&
                    (requires {
                      { alloc != other.alloc } -> std::convertible_to<bool>;
                    } && (alloc != other.alloc))) {
        destroy_deallocate();
        alloc = other.alloc;
        if (cap < other.sz) {
          elems = mystd::allocator_traits<Allocator>::allocate(alloc, other.sz);
          cap = other.sz;
        }
      } else if (other.sz > cap) {
        if (elems)
          destroy_deallocate();
        elems = mystd::allocator_traits<Allocator>::allocate(alloc, other.sz);
        cap = other.sz;
      } else if constexpr (!std::is_trivially_destructible_v<T>)
        for (std::size_t i = other.sz; i < sz; ++i)
          mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
      if constexpr (std::is_trivially_copyable_v<T>)
        std::memcpy(elems, other.elems, other.sz * sizeof(T));
      else {
        std::size_t i = 0;
        try {
          for (; i < other.sz; ++i) {
            if (i < sz)
              elems[i] = other.elems[i];
            else
              mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                            other.elems[i]);
          }
        } catch (...) {
          for (std::size_t j = 0; j < i; ++j)
            mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
          throw;
        }
      }
      sz = other.sz;
    }
    return *this;
  }

  constexpr void assign(std::size_t count, const T &value) {
    if (count > cap) {
      destroy_deallocate();
      sz = 0;
      elems = mystd::allocator_traits<Allocator>::allocate(alloc, count);
      cap = count;
    }
    if constexpr (std::is_trivially_copyable_v<T>) {
      std::fill_n(elems, count, value);
    } else {
      std::size_t min_size = std::min(sz, count);
      for (std::size_t i = 0; i < min_size; ++i)
        elems[i] = value;
      for (std::size_t i = min_size; i < count; ++i)
        mystd::allocator_traits<Allocator>::construct(alloc, elems + i, value);
      if (sz > count)
        for (std::size_t i = count; i < sz; ++i)
          mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
    }
    sz = count;
  }

  template <std::input_iterator InputIt>
  constexpr void assign(InputIt first, InputIt last) {
    if constexpr (std::forward_iterator<InputIt>) {
      std::size_t count = static_cast<std::size_t>(std::distance(first, last));
      if (count > cap) {
        destroy_deallocate();
        sz = 0;
        elems = mystd::allocator_traits<Allocator>::allocate(alloc, count);
        cap = count;
      }
      if constexpr (std::is_trivially_copyable_v<T>) {
        if constexpr (std::contiguous_iterator<InputIt>) {
          std::memmove(elems, &*first, count * sizeof(T));
        } else {
          mystd::copy(first, last, elems);
        }
      } else {
        std::size_t i = 0;
        for (; i < count && first != last; ++i, ++first) {
          if (i < sz)
            elems[i] = *first;
          else
            mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                          *first);
        }
        if (sz > count)
          for (std::size_t j = count; j < sz; ++j)
            mystd::allocator_traits<Allocator>::destroy(alloc, elems + j);
      }
      sz = count;
    } else {
      clear();
      for (; first != last; ++first)
        push_back(*first);
    }
  }

  constexpr void reserve(std::size_t new_cap) {
    if (new_cap <= cap)
      return;
    if (new_cap > max_size())
      throw std::length_error("vector");
    T *new_elems = mystd::allocator_traits<Allocator>::allocate(alloc, new_cap);
    if (elems) {
      if constexpr (std::is_trivially_copyable_v<T>) {
        std::memcpy(new_elems, elems, sz * sizeof(T));
      } else if (std::is_nothrow_move_constructible_v<T> ||
                 !std::is_copy_constructible_v<T>) {
        transfer_elements<true>(new_elems, sz);
      } else {
        transfer_elements<false>(new_elems, sz);
      }
      destroy_deallocate();
    }
    elems = new_elems;
    cap = new_cap;
  }

  constexpr std::size_t capacity() const noexcept { return cap; }

  constexpr void shrink_to_fit() {
    if (cap != sz) {
      if (sz == 0)
        destroy_deallocate();
      else {
        T *new_elems = mystd::allocator_traits<Allocator>::allocate(alloc, sz);
        if constexpr (std::is_trivially_copyable_v<T>) {
          std::memcpy(new_elems, elems, sz * sizeof(T));
        } else if (std::is_nothrow_move_constructible_v<T> ||
                   !std::is_copy_constructible_v<T>) {
          transfer_elements<true>(new_elems, sz);
        } else {
          transfer_elements<false>(new_elems, sz);
        }
        destroy_deallocate();
        elems = new_elems;
        cap = sz;
      }
    }
  }

  constexpr void clear() noexcept {
    if constexpr (!std::is_trivially_destructible_v<T>)
      for (std::size_t i = 0; i < sz; ++i)
        mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
    sz = 0;
  }

  constexpr iterator insert(const_iterator pos, const T &value) {
    std::size_t index = pos - elems;
    if (index > sz) {
      if (index >= cap)
        reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, index + 1));
      for (std::size_t i = sz; i < index; ++i) {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + i);
      }
      sz = index;
    } else if (sz >= cap)
      reserve(sz ? sz * _MYSTD_VECTOR_GROW : 1);
    if (index < sz) {
      if constexpr (std::is_trivially_copyable_v<T>) {
        std::memmove(elems + index + 1, elems + index,
                     (sz - index) * sizeof(T));
        elems[index] = value;
      } else {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + sz,
                                                      std::move(elems[sz - 1]));
        for (std::size_t i = sz - 1; i > index; --i)
          elems[i] = std::move(elems[i - 1]);
        elems[index] = value;
      }
    } else {
      mystd::allocator_traits<Allocator>::construct(alloc, elems + index,
                                                    value);
    }
    ++sz;
    return elems + index;
  }

  constexpr iterator insert(const_iterator pos, std::size_t count,
                            const T &value) {
    if (count == 0)
      return const_cast<iterator>(pos);
    std::size_t index = pos - elems;
    if (index > sz) {
      if (index >= cap)
        reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, index + count));
      for (std::size_t i = sz; i < index; ++i) {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + i);
      }
      sz = index;
    }
    if (sz + count > cap)
      reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, sz + count));
    if constexpr (std::is_trivially_copyable_v<T>) {
      std::memmove(elems + index + count, elems + index,
                   (sz - index) * sizeof(T));
      std::fill_n(elems + index, count, value);
    } else {
      for (std::size_t i = sz + count - 1; i >= index + count; --i) {
        if (i < sz)
          elems[i] = std::move(elems[i - count]);
        else
          mystd::allocator_traits<Allocator>::construct(
              alloc, elems + i, std::move(elems[i - count]));
      }
      for (std::size_t i = index; i < index + count; ++i) {
        if (i < sz)
          elems[i] = value;
        else
          mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                        value);
      }
    }
    sz += count;
    return elems + index;
  }

  template <std::input_iterator InputIt>
  constexpr iterator insert(const_iterator pos, InputIt first, InputIt last) {
    std::size_t index = pos - elems;
    if (index > sz) {
      if constexpr (std::forward_iterator<InputIt>) {
        std::size_t count =
            static_cast<std::size_t>(std::distance(first, last));
        if (index >= cap)
          reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, index + count));
      } else {
        if (index >= cap)
          reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, index + 1));
      }
      for (std::size_t i = sz; i < index; ++i) {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + i);
      }
      sz = index;
    }
    if constexpr (std::forward_iterator<InputIt>) {
      std::size_t count = static_cast<std::size_t>(std::distance(first, last));
      if (count == 0)
        return const_cast<iterator>(pos);
      if (sz + count > cap)
        reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, sz + count));
      if constexpr (std::is_trivially_copyable_v<T>) {
        std::memmove(elems + index + count, elems + index,
                     (sz - index) * sizeof(T));
        if constexpr (std::contiguous_iterator<InputIt>)
          std::memmove(elems + index, &*first, count * sizeof(T));
        else
          mystd::copy(first, last, elems + index);
      } else {
        for (std::size_t i = sz + count - 1; i >= index + count; --i) {
          if (i < sz)
            elems[i] = std::move(elems[i - count]);
          else
            mystd::allocator_traits<Allocator>::construct(
                alloc, elems + i, std::move(elems[i - count]));
        }
        std::size_t i = index;
        for (; i < index + count && first != last; ++i, ++first) {
          if (i < sz)
            elems[i] = *first;
          else
            mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                          *first);
        }
      }
      sz += count;
      return elems + index;
    } else {
      vector<T, Allocator> tmp(first, last, alloc);
      return insert(pos, tmp.begin(), tmp.end());
    }
  }

  template <class Arg>
  constexpr iterator emplace(const_iterator pos, Arg &&arg) {
    std::size_t index = pos - elems;
    if (index > sz) {
      if (index >= cap)
        reserve(std::max(sz ? sz * _MYSTD_VECTOR_GROW : 1, index + 1));
      for (std::size_t i = sz; i < index; ++i) {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + i);
      }
      sz = index;
    } else if (sz >= cap)
      reserve(sz ? sz * _MYSTD_VECTOR_GROW : 1);
    if (index == sz) {
      if constexpr (std::is_trivially_move_constructible_v<T>)
        elems[sz] = T(std::forward<Arg>(arg));
      else
        mystd::allocator_traits<Allocator>::construct(alloc, elems + sz,
                                                      std::forward<Arg>(arg));
    } else {
      if constexpr (std::is_trivially_copyable_v<T>) {
        std::memmove(elems + index + 1, elems + index,
                     (sz - index) * sizeof(T));
        elems[index] = T(std::forward<Arg>(arg));
      } else {
        mystd::allocator_traits<Allocator>::construct(alloc, elems + sz,
                                                      std::move(elems[sz - 1]));
        for (std::size_t i = sz - 1; i > index; --i)
          elems[i] = std::move(elems[i - 1]);
        elems[index] = T(std::forward<Arg>(arg));
      }
    }
    ++sz;
    return elems + index;
  }

  constexpr iterator erase(const_iterator pos) {
    std::size_t index = pos - elems;
    if (index >= sz)
      return elems + sz;
    if constexpr (std::is_trivially_copyable_v<T>) {
      std::memmove(elems + index, elems + index + 1,
                   (sz - index - 1) * sizeof(T));
    } else {
      for (std::size_t i = index; i < sz - 1; ++i)
        elems[i] = std::move(elems[i + 1]);
      mystd::allocator_traits<Allocator>::destroy(alloc, elems + sz - 1);
    }
    --sz;
    return elems + index;
  }

  constexpr iterator erase(const_iterator first, const_iterator last) {
    if (first == last)
      return const_cast<iterator>(first);
    std::size_t index = first - elems;
    std::size_t end_index = last - elems;
    if (index >= sz)
      return elems + sz;
    if (end_index > sz)
      end_index = sz;
    std::size_t count = end_index - index;
    if constexpr (std::is_trivially_copyable_v<T>) {
      std::memmove(elems + index, elems + index + count,
                   (sz - index - count) * sizeof(T));
    } else {
      for (std::size_t i = index; i + count < sz; ++i)
        elems[i] = std::move(elems[i + count]);
      for (std::size_t i = sz - count; i < sz; ++i)
        mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
    }
    sz -= count;
    return elems + index;
  }

  constexpr void push_back(const T &value) {
    if (sz >= cap)
      reserve(sz ? sz * _MYSTD_VECTOR_GROW : 1);
    if constexpr (std::is_trivially_copyable_v<T>)
      elems[sz++] = value;
    else {
      mystd::allocator_traits<Allocator>::construct(alloc, elems + sz, value);
      ++sz;
    }
  }

  constexpr void push_back(T &&value) {
    if (sz >= cap)
      reserve(sz ? sz * _MYSTD_VECTOR_GROW : 1);
    if constexpr (std::is_trivially_move_constructible_v<T>)
      elems[sz++] = std::move(value);
    else {
      mystd::allocator_traits<Allocator>::construct(alloc, elems + sz,
                                                    std::move(value));
      ++sz;
    }
  }

  template <class Arg> constexpr T &emplace_back(Arg &&arg) {
    if (sz >= cap)
      reserve(sz ? sz * _MYSTD_VECTOR_GROW : 1);
    if constexpr (std::is_trivially_move_constructible_v<T>)
      elems[sz] = T(std::forward<Arg>(arg));
    else
      mystd::allocator_traits<Allocator>::construct(alloc, elems + sz,
                                                    std::forward<Arg>(arg));
    return elems[sz++];
  }

  constexpr void pop_back() {
    --sz;
    if constexpr (!std::is_trivially_destructible_v<T>)
      mystd::allocator_traits<Allocator>::destroy(alloc, elems + sz);
  }

  constexpr void resize(std::size_t new_size) {
    if (new_size < sz) {
      if constexpr (!std::is_trivially_destructible_v<T>)
        for (std::size_t i = new_size; i < sz; ++i)
          mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
      sz = new_size;
      return;
    }
    if (new_size > cap)
      reserve(new_size);
    if (new_size > sz) {
      if constexpr (std::is_trivially_default_constructible_v<T>) {
        std::uninitialized_default_construct(elems + sz, elems + new_size);
      } else if constexpr (std::is_default_constructible_v<T>) {
        for (std::size_t i = sz; i < new_size; ++i)
          mystd::allocator_traits<Allocator>::construct(alloc, elems + i);
      } else {
        throw std::logic_error("vector");
      }
      sz = new_size;
    }
  }

  constexpr void resize(std::size_t new_size, const T &value) {
    if (new_size < sz) {
      if constexpr (!std::is_trivially_destructible_v<T>)
        for (std::size_t i = new_size; i < sz; ++i)
          mystd::allocator_traits<Allocator>::destroy(alloc, elems + i);
      sz = new_size;
      return;
    }
    if (new_size > cap)
      reserve(new_size);
    if (new_size > sz) {
      if constexpr (std::is_trivially_default_constructible_v<T>) {
        std::uninitialized_fill(elems + sz, elems + new_size, value);
      } else {
        for (std::size_t i = sz; i < new_size; ++i)
          mystd::allocator_traits<Allocator>::construct(alloc, elems + i,
                                                        value);
      }
      sz = new_size;
    }
  }

  constexpr void swap(vector &other) noexcept(
      mystd::allocator_traits<Allocator>::propagate_on_container_swap::value ||
      mystd::allocator_traits<Allocator>::is_always_equal::value) {
    if constexpr (mystd::allocator_traits<
                      Allocator>::propagate_on_container_swap::value)
      mystd::swap(alloc, other.alloc);
    mystd::swap(elems, other.elems);
    mystd::swap(sz, other.sz);
    mystd::swap(cap, other.cap);
  }
};

\sssc{總論}
本專題於多個核心元件中實作必要且具語意的 operator overloading。各 operator 之實作細節、語法限制與實際使用範例，已於前述章節中分別說明；以下僅就整體設計理念與實務層面的效益進行總結性說明。

透過適當且具節制的 operator overloading，可達成下列目標：
\begin{itemize}
\item 符合 C++20 標準對型別行為與語意的一致性期待，例如比較運算子與三向比較運算子（\verb|operator<=>|）。
\item 與 STL 及現代 C++ idioms 保持一致，使自製容器或型別能自然地套用既有演算法（如 \verb|mystd::qsort|、\verb|std::sort|、\verb|std::ranges|）。
\item 提升程式碼的可讀性與表意性，避免以顯式函式呼叫取代本質上屬於「運算」或「比較」的語意行為。
\item 減少呼叫端的樣板程式碼（boilerplate），使使用者端程式碼更貼近問題本身的抽象層次。
\item 滿足 Structured Bindings 與 ADL（Argument-Dependent Lookup）等語言機制對介面的需求，相關說明請參見 \hyperref[bindings]{\textbf{Structured Bindings}} 與 \hyperref[adl]{\textbf{Argument-Dependent Lookup (ADL)}}。
\item 配合物件生命週期與資源管理需求，遵循 \hyperref[rule]{\textbf{Rule of Three / Five / Zero}}，確保型別在複製、移動與指派語意上的一致性與安全性。
\end{itemize}

相對地，若缺乏必要的運算子支援，呼叫端往往需以輔助函式或顯式介面取代，導致程式碼冗長、語意分散，且不利於與標準函式庫及泛型演算法整合。

至於資料結構與演算法層面的設計與巧思，將與其他元件一併闡述，請參見 \hyperref[2-D-1-1]{\textbf{2-D-1-1 演算法設計}} 與 \hyperref[2-D-1-2]{\textbf{2-D-1-2 程式執行效率優化}}。

\sssc{Static Initialization Order Fiasco (SIOF)}
\label{siof}
In C++, the initialization order of non-local static objects is governed by the following rules:
\begin{itemize}
\item \textbf{Within a single translation unit:} Non-local static objects are initialized in the order of their definitions within that translation unit.
\item \textbf{Across different translation units:} The initialization order of non-local static objects defined in different translation units is unspecified.
\end{itemize}
As a result, if a static object in one translation unit depends on another static object defined in a different translation unit during its initialization, there is no guarantee that the depended-on object has already been initialized. Accessing such an object before its initialization leads to undefined behavior.

To mitigate or avoid SIOF, the following design strategies are generally recommended:
\begin{itemize}
\item Prefer \verb|constexpr| or \verb|inline| variables where possible, as their initialization is guaranteed at compile time or follows well-defined rules.
\item Use function-local static objects, whose initialization is guaranteed to occur upon first use and is thread-safe since C++11.
\item Avoid cross-translation-unit dependencies between non-local static objects, especially during their initialization phase.
\end{itemize}
\sssc{型別層級（type-level）的工具函式}  
在 \verb|include/allocator.hpp| 中，\verb|allocator_traits| 將所有操作（\verb|allocate|、\verb|deallocate|、\verb|construct|、\verb|destroy|、\verb|max_size| 等）設計為 static member function。此設計符合 C++20 標準，其核心意義包括：
\begin{itemize}
  \item 行為取決於 \verb|Alloc| 型別本身，而非 traits 物件實例。
  \item 避免建立不必要的 traits 實體，降低語意與實作負擔。
  \item 可在編譯期透過 SFINAE 或 \verb|if constexpr| 檢測 allocator 是否提供特定介面。請參見 \hyperref[sfinae]{\textbf{Substitution Failure Is Not An Error (SFINAE)}} 與 \hyperref[constexpr]{\textbf{constexpr function}}。
\end{itemize}

\sssc{Substitution Failure Is Not An Error (SFINAE)}
\label{sfinae}
SFINAE is a compile-time technique in template programming. If substituting template parameters results in an invalid type or expression, the compiler silently ignores that overload instead of producing an error. This mechanism enables selective template instantiation and overload resolution.
\sssc{Concepts}
\lable{concepts}
C++20 introduces \textbf{concepts}, which allow explicit constraints on template parameters in a readable and expressive way. Concepts replace some uses of SFINAE with clearer syntax. Key benefits include:
\begin{itemize}
    \item Clear compile-time diagnostics.
    \item Explicit template constraints.
    \item Improved code readability and maintainability.
\end{itemize}
In my STL implementation, concepts are heavily used to determine the appropriate overload or \texttt{if constexpr} branch. This technique enables optimizations for trivially copyable types, contiguous iterators, allocator traits-based optimization, etc.

One example is demonstrated in \textbf{2-B-1-2 指標成員處理} 中的 \hyperref[vector-list]{\textbf{include/vector.hpp}} and described in its \hyperref[vector-describe]{\textbf{敘述}}.

Another example in \verb|include/algorithm-copy.hpp| is described as follows. The file provides three basic data transfer algorithms: \verb|copy|, \verb|copy_if|, and \verb|copy_backward|, mimicking the interface design of C++20 \verb|<algorithm>|. The copy function family demonstrates the polymorphism programming techniques, particularly conceptual parameter constraints and conditional optimization strategies.

I utilize concepts for constraints, such as \verb|std::input_iterator|, \verb|std::weakly_incrementable|, \verb|std::bidirectional_iterator|, and \verb|std::indirectly_writable| in the definition, and
\verb|std::contiguous_iterator| and \verb|std::is_trivially_copyable_v| for specialized acceleration.

In \verb|copy| algorithms, I use \verb|std::memcpy| when possible and element-wise assignment only when necessary, as demonstrated below (line~7--32)
\begin{lstlisting}[language=C++]
template <std::input_iterator InputIt, std::weakly_incrementable OutputIt>
  requires std::indirectly_writable<OutputIt, std::iter_value_t<InputIt>>
constexpr OutputIt copy(InputIt first, InputIt last, OutputIt d_first) {
  if constexpr (std::contiguous_iterator<InputIt> &&
                std::contiguous_iterator<OutputIt> &&
                std::is_trivially_copyable_v<
                    typename std::iterator_traits<InputIt>::value_type> &&
                std::is_same_v<
                    typename std::iterator_traits<InputIt>::value_type,
                    typename std::iterator_traits<OutputIt>::value_type>) {
    if (first == last)
      return d_first;
    const auto *src = std::to_address(first);
    const auto *src_end = std::to_address(last);
    auto *dest = std::to_address(d_first);
    const std::size_t count = (src_end - src) * sizeof(*src);
    std::memcpy(dest, src, count);
    return d_first + (last - first);
  }
  while (first != last) {
    *d_first = *first;
    ++first;
    ++d_first;
  }
  return d_first;
}
\end{lstlisting}

Similarly, in \verb|copy_backward| algorithm, \verb|std::memmove| is used when possible, as demonstrated below (line~49--73)
\begin{lstlisting}[language=C++]
template <std::bidirectional_iterator BidirIt1,
          std::bidirectional_iterator BidirIt2>
constexpr BidirIt2 copy_backward(BidirIt1 first, BidirIt1 last,
                                 BidirIt2 d_last) {
  using ValueType1 = typename std::iterator_traits<BidirIt1>::value_type;
  using ValueType2 = typename std::iterator_traits<BidirIt2>::value_type;
  if constexpr (std::contiguous_iterator<BidirIt1> &&
                std::contiguous_iterator<BidirIt2> &&
                std::is_trivially_copyable_v<ValueType1> &&
                std::is_same_v<ValueType1, ValueType2>) {
    auto n = last - first;
    if (n > 0) {
      std::memmove(std::addressof(*d_last) - n, std::addressof(*first),
                   n * sizeof(ValueType1));
    }
    return d_last - n;
  } else {
    while (first != last) {
      --last;
      --d_last;
      *d_last = *last;
    }
    return d_last;
  }
}

\sssc{Empty Base Optimization (EBO)}

In C++, an empty class normally has size 1 to ensure unique addresses. EBO is a compiler optimization that allows empty base classes to occupy no additional storage when used as a base. In my implementation, EBO is applied in \verb|_tuple_impl|, the internal recursive inheritance mechanism for \verb|mystd::tuple|.
\sssc{Template Compile-Time Recursion}
\label{recursion}
Templates can perform computation at compile time using recursion:
\begin{itemize}
    \item Each template instantiation computes one step of the recursion.
    \item Recursion terminates at a base case specialization.
    \item Computation occurs entirely at compile time, producing constant expressions.
\end{itemize}
This technique is used in \verb|include/tuple.hpp| for \verb|_tuple_impl|, \verb|tuple|, and related non-member functions.
\sssc{std::declval template}
\label{declval}
\texttt{std::declval<T>()} defined in \texttt{<utility>} is a template intended exclusively for use in unevaluated contexts (such as within \texttt{decltype}, \texttt{sizeof}, or SFINAE expressions). It provides a fictional value of type \texttt{T} or \texttt{T\&} without requiring that the type be constructible. This enables compile-time type deduction even for non-instantiable or non-default-constructible types.

\texttt{std::declval} is indispensable in implementing traits and SFINAE-based detection idioms.
\sssc{Ref-Qualified Member Functions}
\label{ref-qualified}
C++11 introduced \textbf{ref-qualified} member functions. These allow a class to distinguish overloads based on whether the object is an lvalue (\texttt{\&}) or an rvalue (\texttt{\&\&}). For example:
\begin{lstlisting}[language=C++]
template <class T>
struct S {
    T data;
    T& get() & { return data; }
    T&& get() && { return std::move(data); }
};
\end{lstlisting}
This mechanism is widely used in the member functions overloads in STL to control move semantics.
\sssc{decltype}
\label{decltype}
\texttt{decltype} is a compile-time operator that deduces the declared type of an expression without performing the expression's evaluation. Unlike \verb|auto|, which uses assignment-like deduction rules, \texttt{decltype} preserves the exact type, including references and cv-qualifiers.

Given an expression \verb|e|, the type deduced by \texttt{decltype(e)} depends on the
expression category.
\begin{itemize}
    \item If \verb|e| is a unparenthesized id-expression naming a variable, then \texttt{decltype(e)} yields the declared type of the variable. Note that a parenthesized one is a lvalue expression.
    \item If \verb|e| is an \emph{lvalue} expression, \texttt{decltype(e)} yields \verb|T\&|.
    \item If \verb|e| is an \emph{xvalue}, \texttt{decltype(e)} yields \verb|T&&|.
    \item If \verb|e| is a \emph{prvalue}, \texttt{decltype(e)} yields \verb|T|.
\end{itemize}

\verb|decltype(auto)| preserves the exact type of the initializer.

\texttt{decltype} is crucial in producing type traits or meta-functions in template metaprogramming in many templates in my implementation, and \verb|decltype(auto)| is also used together with perfect forwarding and template overloads in my implementation.
\sssc{Curiously Recurring Template Pattern (CRTP)}
The Curiously Recurring Template Pattern (CRTP) is a C++ idiom in which a class Derived inherits from a class template Base that takes Derived itself as a template parameter.

CRTP allows classes to achieve polymorphic behavior at compile time, combining interface abstraction, code reuse, and high performance. It is particularly suitable when runtime overhead of virtual functions is undesirable or when the set of derived classes is fixed at compile time.

\sssc{constexpr function}
\label{constexpr}
The constexpr specifier is introduced in C++11, and declares that it is possible to evaluate the value of the entities at compile time. Such entities can then be used where only compile time constant expressions are allowed (provided that appropriate function arguments are given).

Since C++20, many functions in STL is constexpr, notably many of the constructors and member functions of most containers. The list below is the specification of constexpr function in C++20, the standard my implementation complies with.
\bit
\item A constexpr specifier used in the first declaration of a function or static data member implies inline. If any declaration of a function or function template has a constexpr specifier, then every declaration must contain that specifier.
\item A function or function template can be declared constexpr. A function is constexpr-suitable if all following conditions are satisfied:
\bit
\item If it is a constructor or destructor, its class does not have any virtual base class. 
\item Its return type (if exists) is a literal type.
\item Each of its parameter types is a literal type.
\item It is not a coroutine.
\item Its function body is = default, = delete, or does not enclose the following: 
\bit
\item goto statements
\item statements with labels other than case and default 
\item definitions of variables of non-literal types
\item definitions of variables of static or thread storage duration 
\eit
\item Except for instantiated constexpr functions, non-templated constexpr functions must be constexpr-suitable.
\item For a non-constructor constexpr function that is neither defaulted nor templated, if no argument values exist such that an invocation of the function could be an evaluated subexpression of a core constant expression, the program is ill-formed, no diagnostic required.
\item For a templated constexpr function, if no specialization of the function/class template would make the templated function constexpr-suitable when considered as a non-templated function, the program is ill-formed, no diagnostic required.
\item An invocation of a constexpr function in a given context produces the same result as an invocation of an equivalent non-constexpr function in the same context in all respects, with the following exceptions:
\bit
\item An invocation of a constexpr function can appear in a constant expression.
\item Copy elision is not performed in a constant expression. 
\eit
\item constexpr constructor: On top of the requirements of constexpr functions, a constructor also needs to satisfy all following conditions to be constexpr-suitable:
\bit
\item Its function body is = delete or satisfies the following additional requirements: 
\bit
\item Every non-variant non-static data member and base class subobject is initialized.
\item If the constructor is a delegating constructor, the target constructor is a constexpr constructor.
\item If the constructor is a non-delegating constructor, every constructor selected to initialize non-static data members and base class subobjects is a constexpr constructor.
\eit
\item The class does not have any virtual base class. 
\item For a constexpr constructor that is neither defaulted nor templated, if no argument values exist such that an invocation of the function could be an evaluated subexpression of the initialization full-expression of some object subject to constant expression, the program is ill-formed, no diagnostic required.
\item constexpr destructor: On top of the requirements of constexpr functions, a destructor also needs to satisfy all following conditions to be constexpr-suitable:
\bit
\itrm For every subobject of class type or (possibly multi-dimensional) array thereof, that class type has a constexpr destructor.
\item The class does not have any virtual base class. 
\eit

\sssc{Named Return Value Optimization (NRVO)}
Named Return Value Optimization (NRVO) is a compiler optimization that eliminates unnecessary copy or move constructions when returning a local object from a function.

If a function returns a named local variable matching the function's return type directly without parenthesis, the compiler is allowed to construct the object directly in the caller's storage, bypassing temporary objects.
\sssc{no\_unique\_address attribute}
\label{no-unique}
Introduced in C++20, \texttt{[[no\_unique\_address]]} attribute allows empty member objects to be optimized similarly to empty bases, reducing storage overhead. It is used for allocator members in my STL containers.
