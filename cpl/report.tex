\ssc*{2-A-1-1 描述 class 之間的關係}
\sssc{include/tuple.hpp}
\begin{itemize}
\item \verb|template <typename T> struct _is_tuple| inherits from \verb|std::false_type|, serving as a primary type trait to determine whether a given type is a tuple. (line~20)
\item \verb|template <std::size_t I, class Head, class... Tail> struct _tuple_impl<I, Head, Tail...>| publicly inherits from \verb|_tuple_impl<I + 1, Tail...>|, forming a recursive inheritance structure that represents each tuple element at a distinct level. (line~54)
\item \verb|template <std::size_t I, class Head, class... Tail> struct _tuple_impl<I, Head, Tail...>| contains a data member \verb|Head head;|, demonstrating class composition where each level stores exactly one tuple element. (line~56)
\item \verb|template <class... Types> class tuple| composes an internal implementation object \verb|_tuple_impl<0, Types...> _impl|, separating the public interface from the recursive storage implementation. (line~258)
\item \verb|template <typename... Ts> struct _is_tuple<mystd::tuple<Ts...>>| inherits from \verb|std::true_type|, providing a specialization that marks \verb|mystd::tuple| as a valid tuple type at compile time. (line~578)
\item \verb|template <class T> struct _count_of_type<T>| inherits from \verb|std::integral_constant<std::size_t, 0>|, defining the recursion base case for counting occurrences of a type within a parameter pack. (line~603)
\item \verb|template <class T, class Head, class... Tail> struct _count_of_type<T, Head, Tail...>| inherits from \verb|std::integral_constant<std::size_t, (std::is_same_v<T, Head> ? 1 : 0) + _count_of_type<T, Tail...>::value>|, implementing a recursive compile-time algorithm via inheritance. (line~606)
\item \verb|template <class T, class... Tail> struct _index_of_type<T, T, Tail...>| inherits from \verb|std::integral_constant<std::size_t, 0>|, representing the base case for determining the index of a type within the tuple. (line~618)
\item \verb|template <class T, class Head, class... Tail> struct _index_of_type<T, Head, Tail...>| inherits from \verb|std::integral_constant<std::size_t, 1 + _index_of_type<T, Tail...>::value>|, forming a recursive inheritance chain to compute the type index at compile time. (line~622)
\item \verb|template <class... Types, class Alloc> struct uses_allocator<std::tuple<Types...>, Alloc>| inherits from \verb|std::true_type|, indicating that \verb|mystd::tuple| participates in allocator-aware construction semantics. (line~732)
\item \verb|template <class... Types> struct tuple_size<mystd::tuple<Types...>>| inherits from \verb|std::integral_constant<std::size_t, sizeof...(Types)>|, integrating \verb|mystd::tuple| with the standard tuple interface. (line~739)
\item \verb|template <std::size_t I, class Head, class... Tail> struct tuple_element<I, mystd::tuple<Head, Tail...>>| inherits from \verb|tuple_element<I - 1, mystd::tuple<Tail...>>|, forming a recursive inheritance structure to map indices to element types. (line~748)
\end{itemize}

\ssc*{2-A-2-1 繼承關係}
Each specialization corresponds to one logical tuple element, where the template parameter \verb|I| denotes the zero-based index of the current element. The inheritance chain terminates at a base specialization with an empty parameter pack. This design decomposes the tuple into a linear sequence of base classes, enabling element access, type computation, and index-based operations to be resolved entirely at compile time without runtime overhead.
