\ssc{2-A-1-2 畫出 class 之間的關係圖}
\sssc{include/tuple.hpp}
Each specialization corresponds to one logical tuple element, where the template parameter \verb|I| denotes the zero-based index of the current element. The inheritance chain terminates at a base specialization with an empty parameter pack. This design decomposes the tuple into a linear sequence of base classes, enabling element access, type computation, and index-based operations to be resolved entirely at compile time without runtime overhead.

Inheritance is used deliberately as a compile-time recursion mechanism. Each instantiation step corresponds to a UML generalization, while the recursion unfolds as a deterministic compile-time sequence. UML sequence diagrams are therefore appropriate to visualize template instantiation order.
\begin{plantuml}
@startuml

' Base empty _tuple_impl
class "_tuple_impl<I>" as TupleImplEmpty {
  +template <std::size_t J> element_type = void
  +_tuple_impl()
  +swap(other)
}

' Recursive _tuple_impl
class "_tuple_impl<I, Head, Tail...>" as TupleImpl {
  +Head head
  +template <std::size_t J> get()
  +swap(other)
}

' tuple class
class "tuple<Types...>" as Tuple {
  +_tuple_impl<0, Types...> _impl
  +get<I>()
  +swap(other)
}

' Type traits and helpers (optional)
class "_is_tuple<T>" as IsTuple {
  +inherits std::false_type
}
class "_is_tuple<mystd::tuple<Ts...>>" as IsTupleSpecialized {
  +inherits std::true_type
}

class "_count_of_type<T, Types...>" as CountType
class "_index_of_type<T, Types...>" as IndexType

' Inheritance / composition
TupleImpl <|-- TupleImplEmpty
TupleImplEmpty <|-- TupleImpl
TupleImpl --> Tuple : composition
Tuple --> IsTupleSpecialized : uses
IsTupleSpecialized <|-- IsTuple

@enduml
